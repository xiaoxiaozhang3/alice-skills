# Skill Judge Checklist

Use after editing a skill.

- [ ] `name` is lowercase hyphen-case
- [ ] `description` starts with or behaves like "Use when..."
- [ ] `description` describes triggers, not the whole workflow
- [ ] SKILL.md is concise enough to load comfortably
- [ ] Heavy details are moved to references
- [ ] Non-obvious branching has a decision tree
- [ ] Negative constraints include alternatives
- [ ] Output format is explicit
- [ ] Self-check catches likely failures
- [ ] At least two should-trigger cases exist
- [ ] At least one should-not-trigger case exists
