# Document Decisions

## Purpose

Turn one-off work into durable project memory and future skill improvements.

## Source Ideas

`documentation-and-adrs`, `shipping-artifacts`, OpenSpec archive.

## Process

1. Read the completed change folder.
2. Summarize the outcome and decisions.
3. Record why alternatives were rejected.
4. Capture verification evidence and remaining risks.
5. Write lessons to `08-archive.md`.
6. If a lesson applies broadly, propose a skill or workflow update.

## What To Preserve

- Architecture decisions
- API/data contracts
- Security and permission assumptions
- Test and verification strategy
- Known trade-offs
- Follow-up improvements

## Self-Check

- [ ] Archive explains why, not only what
- [ ] Reusable lesson is separated from task-specific detail
- [ ] Future reader can resume context from files
