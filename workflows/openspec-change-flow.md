# OpenSpec-Style Change Flow

Each meaningful change gets a folder. The folder is the durable record.

## Phases

1. `00-change.md` - identify the change, source request, scope, and constraints
2. `01-requirements.md` - clarify user intent and acceptance criteria
3. `02-design.md` - choose an approach and record trade-offs
4. `03-plan.md` - turn design into ordered implementation tasks
5. `04-tasks.md` - executable checklist with exact verification points
6. `05-progress.md` - current checkpoint and resume state
7. `06-verification.md` - command evidence and manual verification notes
8. `07-review.md` - spec compliance, quality, risk, and follow-ups
9. `08-archive.md` - lessons, decisions, and reusable skill improvements

## Context Hygiene

Move between phases by reading artifacts, not by relying on conversation memory. When context gets large, summarize state into `05-progress.md` and continue from files.
